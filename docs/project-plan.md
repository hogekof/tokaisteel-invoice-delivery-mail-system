# プロジェクト進行計画

## 概要

請求書・納品書メール送信システムの開発プロジェクト進行計画。
お客様（東海鋼材工業様）との打ち合わせ・確認を軸にしたマイルストーン。

## ステップ

### ステップ1：要件定義・環境準備（←いまここ）

- 要件定義の打ち合わせ（オンライン）
  - 現在の請求書・納品書の発行業務の流れ
  - 軽技Webの利用状況・データの構成
  - 送信対象の範囲・送信ルール
  - 帳票（請求書・納品書）の現行フォーマット
  - インボイス制度等への対応要否
  - 電子帳簿保存法への対応方針（検索要件は必須。改ざん防止措置の方針確認）
- 軽技Webの環境準備（並行して先行依頼）
  - 接続先情報（IP/URL）の取得
  - 開発用アカウントの発行
  - 弊社IPからのアクセス許可（弊社IP: x.x.x.x）
- 軽技Webへの接続・データ取得検証

### ステップ2：データ・帳票・メールの確認（複数回想定）

- 取得データの項目・内容の確認
- 運用ルールの確認（送信可否、メールアドレス管理）
- 請求書・納品書PDFのレイアウト確認
- メール文面（件名・本文）の確認

### ステップ3：画面デモ・操作フローの確認

- 管理画面を実際に触っていただき、操作感を確認
- 確認待ち→確定→送信の業務フローの最終確認
- 修正・ご要望のフィードバック

### ステップ4：テスト運用

- 実データでの動作確認
- 試験送信（テスト用アドレス→限定数社→全体）
- 問題点の洗い出し・修正対応

### ステップ5：本番稼働・お引き渡し

- 本番環境への切替（月初タイミング推奨）
- 並行運用期間
- 操作説明・お引き渡し
- 稼働後フォローアップ

## 進め方の方針

- 各ステップの間でお客様のご確認・ご判断をいただいた上で次へ進む
- 打ち合わせはオンライン（Zoom等）で実施
- ステップ2の帳票確認は複数回のイテレーションを想定
- テスト運用は段階的に範囲を拡大（誤送信防止）
- 本番切替は月初タイミングで実施（請求締め日との整合）

- [ ] 軽技Webの接続先情報（IP/URL）
- [ ] 軽技Web開発用アカウントの発行
- [ ] 弊社IP（x.x.x.x）からのアクセス許可
- [ ] 要件定義打ち合わせの日程調整

### 以降のステップで必要

- [ ] 会社ロゴ・印影等の画像データ

- [ ] 得意先メールアドレスリストの整備
- [ ] 得意先マスタ備考2への送信可否データ入力
- [ ] メール文面（件名・本文）の承認
