# 備考2フォーマット案

## 概要

得意先マスタの備考2フィールドに、以下の3つの情報を格納する。
- 請求書の送信可否（ON/OFF）
- 納品書の送信可否（ON/OFF）
- メールアドレス（1社1アドレス）

## フォーマット案

1行目にメールアドレス、2行目に請求書・納品書の送信可否を改行区切りで入力する。

```
メールアドレス
請求書ON/OFF,納品書ON/OFF
```

**例:**
| パターン | 備考2の値 | 意味 |
|---------|----------|------|
| 両方送信 | `sample@example.com`<br>`ON,ON` | 請求書○ 納品書○ |
| 請求書のみ | `sample@example.com`<br>`ON,OFF` | 請求書○ 納品書× |
| 納品書のみ | `sample@example.com`<br>`OFF,ON` | 請求書× 納品書○ |
| 送信しない | （空欄） | 両方× |
| 未設定 | （空欄） | デフォルトON扱い（要確認） |

**入力イメージ（備考2フィールド）:**
```
sample@example.com
ON,ON
```

※ON/OFFは大文字・小文字どちらでも対応可能（on, Off, ON すべてOK）
※メールアドレスは現時点では1社1アドレスですが、将来的に複数アドレスが必要になった場合はカンマ区切りで対応可能な設計にしています（例: `aaa@example.com,bbb@example.com`）
※宛名（担当者名）を入れたい場合は以下の形式にも対応可能です:
```
山田 太郎<sample@example.com>
ON,ON
```

## 未設定時のデフォルト動作

| 備考2の状態 | システムの動作 |
|------------|--------------|
| 空欄 | 送信ON（デフォルト）※要確認 |
| メールアドレスなし | 送信ONでもメール送信不可 → エラー通知 |
| フォーマット不正 | 送信ON（デフォルト）※要確認 |

## 確認ポイント

- [ ] このフォーマットで運用可能か
- [ ] 備考2が未設定の得意先をどう扱うか（案: 一覧に「未設定」バッジを表示し、確定不可にする）
- [ ] 備考2の編集はどのシステムで行うか（基幹システムの名称・操作手順）。画面上に「○○で備考2を入力してから再取得してください」とガイダンスを出したい
- [ ] 本システムの操作者（営業担当者）に基幹システムの得意先マスタの編集権限はあるか。ない場合、誰に依頼する運用になるか
- [ ] 備考2にメールアドレスはあるがON/OFFがない場合の扱い（デフォルトON？OFF？）
- [ ] 送信ONだがメールアドレスがない場合の扱い（確定時にエラーにするか、スキップするか）
- [ ] 備考2に既にデータが入っている得意先はあるか（上書きの影響）
- [ ] メールの宛名に担当者名を入れたいか（会社名は得意先マスタから取得可能。担当者名が必要なら備考2に追加が必要）
- [ ] データ整備の期限目安
