# Implementation Plan

## Task 1: プロジェクト基盤とインフラストラクチャのセットアップ
- [ ] 1. プロジェクト基盤とインフラストラクチャのセットアップ

- [ ] 1.1 (P) Pythonプロジェクト構造の初期化
  - pyproject.tomlまたはrequirements.txtで依存関係を定義（Dash、boto3、pydantic、sendgrid、weasyprint、selenium等）
  - Python 3.12環境のセットアップ
  - 環境変数管理の仕組みを構築（.env.exampleを用意）
  - _Requirements: 9.1, 9.2, 9.3, 9.4_

- [ ] 1.2 (P) AWS SAMテンプレートの作成
  - Lambda関数（Dashアプリ用、バッチ処理用）の定義
  - API Gatewayの設定（IP制限リソースポリシー含む）
  - DynamoDBテーブル定義（documents、customers、departments）
  - S3バケット定義（PDFストレージ用）
  - EventBridge Schedulerの設定（毎日10時トリガー）
  - _Requirements: 7.1, 9.1, 9.2_

- [ ] 1.3 DynamoDBテーブルスキーマの実装
  - documentsテーブル: PK(SLIP#{slip_number})、SK(TYPE#{document_type})
  - GSI: department-status-index、customer-fetched_at-index
  - customersテーブル: PK(customer_code)
  - departmentsテーブル: PK(department_id)
  - 条件付き更新用のversion属性を含める
  - _Requirements: 2.3, 3.1, 4.1_

## Task 3: 軽技Webスクレイピングサービスの構築
- [ ] 3. 軽技Webスクレイピングサービスの構築

- [ ] 3.1 ScrapingServiceの実装

  - 部門別の保存済みクエリ実行機能

  - ログイン失敗・タイムアウト等のエラーハンドリング

- [ ] 3.2 得意先マスタ送信可否取得機能の実装

## Task 4: データ取得オーケストレーションサービスの構築
- [ ] 4. DataFetchServiceの実装
  - ScrapingServiceを呼び出してCSVデータを取得
  - CSVを解析して請求書データと納品書データに分類
  - 送信可否デフォルト値を適用
  - **処理予定/処理済みに存在する伝票番号を除外（既存データとの重複排除）**
  - **確認待ちデータはDBに保存しない（メモリ上でUIに返却、古いデータによる誤送信防止）**
  - 処理状態（進行中、完了、失敗）の管理
  - 取得件数（請求書数、納品書数）の集計
  - エラー時のエラーメッセージ生成
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 2.5, 2.6, 2.7, 2.8, 8.1, 8.2, 8.3, 8.4_

## Task 9: APIエンドポイントの構築
- [ ] 9. APIエンドポイントの構築

- [ ] 9.1 データ取得APIの実装
  - POST /api/data-fetch: 部門指定でのデータ取得
  - POST /api/data-fetch/refetch: 指定書類の再取得
  - GET /api/data-fetch/status: 取得処理状態の確認
  - エラーレスポンス（400, 500, 503）の実装
  - _Requirements: 2.1, 2.4, 2.5, 2.6, 2.7, 3.9_

- [ ] 9.2 書類管理APIの実装
  - **確認待ち一覧は /api/data-fetch から取得（メモリ上のデータ）**
  - GET /api/documents/scheduled: 処理予定書類一覧取得（DBから）
  - GET /api/documents/processed: 処理済み書類一覧取得（DBから）
  - GET /api/documents/{id}/detail: 書類詳細・明細取得
  - **POST /api/documents/confirm: 書類確定（メモリ上のデータをDBに保存）**
  - **POST /api/documents/cancel-confirmation: 確定解除（DBから削除）**
  - GET /api/documents/counts: 件数取得（処理予定/処理済みのみ）
